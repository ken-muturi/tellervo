For Hackers

This page has all the things that are most useful if you plan to hack on Corina yourself.

Source Code

Corina source code is kept in the CVS version control system.

To get the Corina source code via anonymous CVS:

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/corina login 
cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/corina co corina

If you're registered as a SourceForge developer and want to make changes, be sure to use "ext" instead of "pserver" in the above commands, and have CVS_RSH set to "ssh". (I have to approve you before you can actually make changes.) You can set the CVSROOT variable instead of using the -d flag, as well.

Bugs

Sourceforge provides a bug database for each project. I don't use the SourceForge bug tracker for my bugs yet, but it might be nice in the future.

  • View bugs

In the past, since my users have always come to me directly to report their bugs, I simply wrote the bugs down in a text file. If I want to run my project out of CVS, I'll need a better system. One idea I have is to have a folder called "Bugs", containing a bunch of text files, one per bug. The idea is that I can add bug-files, and commit them at any time, and fix them by committing the patch (however many files it changes) and removing the bug file at once (or moving it to a "Fixed Bugs" folder for future reference).
